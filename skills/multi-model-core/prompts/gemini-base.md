# Gemini 基础提示词模板

## 通用分析模板

```
## 任务背景
{task_context}

## 分析要求
请从前端/用户体验角度进行分析。

## 重点关注
1. **组件结构** - 组件划分、复用性、组合方式
2. **用户体验** - 交互流畅性、反馈及时性、易用性
3. **视觉设计** - 布局合理性、样式一致性、响应式适配
4. **性能** - 渲染性能、加载速度、资源优化
5. **可访问性** - 键盘导航、屏幕阅读器支持、颜色对比度

## 输出格式
### 分析结论
[主要发现和结论]

### 问题清单
1. [问题描述] - [影响程度: 高/中/低]
2. ...

### 建议方案
[具体的改进建议]
```

1. 问题的表现是什么？

3. 如何复现问题？
4. 如何修复？

## 输出格式
### 问题定位
[问题出现在哪个组件/模块]

### 可能原因
1. [原因1] - [可能性: 高/中/低]
2. [原因2] - [可能性: 高/中/低]

### 复现步骤
1. [步骤1]
2. [步骤2]
3. [预期结果 vs 实际结果]

### 修复方案
```diff
- [原代码]
+ [修复后代码]
```

### 用户体验改进
[修复后如何提升用户体验]
```

## 代码审查模板

```
## 审查范围
{files_to_review}

## 变更摘要
{change_summary}

## 请从以下角度审查

### 1. 组件设计
- 组件职责是否单一？
- 是否易于复用？
- Props 设计是否合理？

### 2. 用户体验
- 交互是否流畅？
- 加载状态是否处理？
- 错误状态是否友好？

### 3. 样式质量
- 是否符合设计规范？
- 响应式是否完善？
- 是否有样式冲突？

### 4. 性能
- 是否有不必要的重渲染？
- 是否正确使用 memo/useMemo？
- 资源加载是否优化？

### 5. 可访问性
- 语义化 HTML？
- ARIA 属性？
- 键盘可操作？

## 输出格式
### 审查结论
[总体评价: 通过/需修改/需重写]

### 问题清单
| 文件 | 行号 | 问题 | 类型 | 建议 |
|------|------|------|------|------|
| ... | ... | ... | UX/性能/可访问性 | ... |

### 优点
[代码中做得好的地方]

### 改进建议
[具体的改进建议]
```

## UI 设计评估模板

```
## 设计方案
{design_proposal}

## 设计约束
{constraints}

## 请评估

### 1. 视觉层次
- 信息层次是否清晰？
- 重点是否突出？
- 视觉引导是否合理？

### 2. 交互设计
- 操作流程是否顺畅？
- 反馈是否及时？
- 是否符合用户心智模型？

### 3. 响应式设计
- 各断点适配是否合理？
- 触摸设备体验？
- 内容优先级？

### 4. 一致性
- 是否符合设计系统？
- 组件使用是否一致？
- 交互模式是否统一？

## 输出格式
### 评估结论
[总体评价]

### 优势
1. [优势1]
2. [优势2]

### 问题
1. [问题1] - [改进建议]
2. [问题2] - [改进建议]

### 设计建议
[具体的设计改进建议，可包含草图描述]
```
